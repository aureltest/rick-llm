# Rick LLM 🧪

Fine-tune **Gemma 4 E4B** into a **French-speaking Rick Sanchez** persona, using QLoRA on a single RTX 5080 (16 GB VRAM). The merged model is pushed to the Hugging Face Hub, then converted to GGUF for local inference via Ollama or LMStudio.

> *"Tu incarnes Rick Sanchez, un scientifique de génie interdimensionnel..."*

## Pipeline

```
Rick & Morty transcripts (EN)
        │
        │  load_dataset.py
        ▼
ShareGPT pairs cleaned + translated to FR (HF Hub)
        │
        │  finetune.ipynb
        ▼
LoRA adapter → merged 4-bit NF4 model (HF Hub)
        │
        │  llama.cpp / convert_hf_to_gguf.py
        ▼
GGUF → Ollama / LMStudio
```

## Requirements

- **GPU**: NVIDIA Blackwell or newer with ≥ 16 GB VRAM (developed on RTX 5080)
- **OS**: WSL2 Ubuntu 22.04+ (or native Linux)
- **CUDA**: 12.8+
- **NVIDIA driver**: 570+ (required for Blackwell `sm_120`)
- **Python**: 3.10+
- Hugging Face account + token (model pushes target a private repo by default)
- (Optional) An OpenAI-compatible LLM endpoint for the dataset cleaning step — defaults to OpenAI, but a local Ollama server works too

## Setup

```bash
git clone https://github.com/<your-user>/rick-llm.git
cd rick-llm

# install deps (use the unsloth instructions matching your CUDA version)
pip install unsloth datasets transformers trl peft bitsandbytes accelerate
pip install openai python-dotenv tqdm wandb

# config
cp .env.example .env
# then fill in the values
```

`.env` keys:

| Variable | Purpose |
|---|---|
| `OPENAI_API_KEY` | Auth for the *teacher* LLM (cleaning/translation). Set to anything if using local Ollama. |
| `HUGGINGFACE_TOKEN` | Push access for the dataset and the merged model. |
| `HF_DATASET_NAME` | Where the cleaned FR ShareGPT dataset is pushed. |
| `HF_MODEL_NAME` | Where the merged 4-bit model is pushed. |
| `LLM_BASE_URL` | OpenAI-compatible base URL for the teacher (e.g. `http://localhost:11434/v1` for Ollama). Leave empty for OpenAI. |
| `LLM_MODEL` | Teacher model id (e.g. `gpt-4o-mini` or `gemma4:e4b`). |

## Usage

### 1. Build the dataset

```bash
python load_dataset.py
```

This:
1. Loads `Prarabdha/Rick_and_Morty_Transcript` from the HF Hub.
2. Builds `(non-Rick line → Rick reply)` ShareGPT pairs within the same episode.
3. Cleans (removes stage directions) and translates each line to French via the *teacher* LLM.
4. Saves to `./dataset_cache/` and pushes to `$HF_DATASET_NAME`.

### 2. Fine-tune

Open [finetune.ipynb](finetune.ipynb) and run it end-to-end. The notebook:
- Loads Gemma 4 E4B in 4-bit and applies the `gemma-4` chat template.
- Trains a LoRA adapter (r=16, α=16, RSLoRA, dropout 0.05) for 2 epochs with `train_on_responses_only`.
- Merges the adapter into the base model and re-quantizes to NF4 at `./merged_model/`.
- Pushes the merged model to `$HF_MODEL_NAME`.

> 💡 If the kernel crashes mid-training, the notebook contains a recovery cell (id `1fad64f7`) that reloads the latest `output/checkpoint-XXX/` and produces `./merged_model/` from a fresh kernel.

Training telemetry is reported to **Weights & Biases** under the run name `rick-gemma4-e4b-fr`.

### 3. Convert to GGUF and run locally

This step is performed outside the repo, with [llama.cpp](https://github.com/ggerganov/llama.cpp):

```bash
python llama.cpp/convert_hf_to_gguf.py ./merged_model --outfile rick-fr.gguf
```

Then load `rick-fr.gguf` in Ollama (via a `Modelfile`) or LMStudio.

## Key design choices

- **Two LLMs, two roles.** A *teacher* (OpenAI or local Ollama) cleans and translates the transcripts at dataset time. The *student* (Gemma 4 E4B) is fine-tuned on the result. Don't conflate `LLM_MODEL` (teacher) with `HF_MODEL_NAME` (student).
- **VRAM-bound, end to end.** Every step is sized for 16 GB: 4-bit base loading, LoRA-only fine-tune with 8-bit AdamW, batch 1 × grad-accum 8, and a final NF4 merge that peaks ~8 GB. Switching to BF16 merge or full fine-tuning won't fit.
- **Multimodal-shaped chat template.** Gemma 4 E4B's processor expects message `content` as a list of typed parts (`[{"type": "text", "text": ...}]`), not a bare string.
- **Canonical system prompt.** `RICK_SYSTEM_PROMPT` in [load_dataset.py](load_dataset.py) is baked into every training pair and re-used at inference. Changing it requires regenerating the dataset.
